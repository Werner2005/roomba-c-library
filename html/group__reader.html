Makro-Dokumentation

#define ROOMBA_NONBLOCKING

Wenn dieses Define definiert ist werden alle Blockiere Methoden nicht blockieren und immer ROOMBA_ERROR_AGAIN zurückgeben wenn keine Daten verfügbar sind.

Define muss über den Compiler gesetzt werden!(-DROOMBA_NONBLOCKING)

#define ROOMBA_SENSOR_BUFFER_SIZE   173

Roomba Senosr Buffer Größe

#define ROOMBA_SHORT_READ

Wenn diese Define gesetzt ist. Wird ein zweiter Ringbuffer vorgeschaltete. Ziel ist die Zeit die Read benötigt zu verringern.

Die Daten werden dann nicht mehr von roomba_read() verarbeitet sondern beim Abrufen der Daten(roomba_hasSensorData() und roomba_getSensorData)

Define muss über den Compiler gesetzt werden!(-DROOMBA_SHORT_READ)

#define ROOMBA_STREAM_IGNORE   50

Anzahl von Stream Paketen die Ignoriert werden sollen falls der Interne Buffer voll ist

Dokumentation der Funktionen

void roomba_abort_sensor ( )

Bricht Sensor Pasing ab

Es kann vorkommen das nicht alle Pakete eines angeforderten Befehles ankommen daher ist es wichtig das auch ein Sensor Befehl abgebrochen werden kann

Falls aktuell kein Sensor Befehl aktiv ist hat dieser Befehl keine Auswirkung!

Diese Funktion setzt den Fehler ROOMBA_ERROR_ABORT_SENSOR

Falls ein Sensor Stream aktiv ist wird dieser mit dem Befehl ROOMBA_STREAM_PAUSE_RESUME beendet

int8_t roomba_read ( uint8_t  c)

Verarbeite ein Zeichen

Parameter
cZeichen das Verarbeitet werden soll
Rückgabe
Kann verschiedene Werte zurückgeben
  • 0 Kein Fehler
  • -ROOMBA_ERROR_BUFFER_FULL(-ROOMBA_ERROR_BUFFER_FULL_CHAR, -ROOMBA_ERROR_BUFFER_FULL_STREAM) Buffer voll
  • -ROOMBA_ERROR_NO_SIGN_EXPECTED Kein Zeichen erwartet Beim verarbeiten eines Stream können folgende (Fehler auftreten Fehler kännen nur auftreten wenn ROOMBA_SHORT_READ nicht gesetzt ist!)
  • -ROOMBA_ERROR_HEADER_NOT_FOUND Zeichen war nicht das erwarte Header Zeichen(19)
  • -ROOMBA_ERROR_SENSORID_NOT_FOUND Senesor ID nicht korrekt
  • -ROOMBA_ERROR_CHECKSUM_NOT_VALID Checksumme nicht korrekt
