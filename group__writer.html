Ausführliche Beschreibung

Dieses Module ist für das schreiben eines Kommandos an den Roomba verantwortlich

Dokumentation der Funktionen

int8_t roomba_sendCommand ( roombaCommand_t *  c)

Sendet ein Kommando

Wenn ein ROOMBA_SENSORS Befehl gesendet wird und derzeit noch ein Sensor Befehl in Arbeit ist.(Außer wenn ROOMBA_NONBLOCKING gesetzt ist!). Dieser Befehl bereitet das System auf den Empfang der Daten vor!

Bei den Befehlen ROOMBA_QUERY_LIST und ROOMBA_STREAM werden alle Gruppen Sensor Daten automatisch gegen ihre echte Werte getauscht! Beispiel: ROOMBA_SENSOR_GROUP_1 gegen: ROOMBA_SENSOR_BUMPS_WHEEL_DROPS, ROOMBA_SENSOR_WALL, ...

Bei dem Befehl ROOMBA_STREAM_PAUSE_RESUME wird bei den Befehl 0(Stream Abruf) Automatische das Interne System Mithilfe von roomba_reset gestopt! Bei 1 wird der Empfang von Stream Daten wieder vorbereitet.

Warnung
Es darf vor dem Befehl ROOMBA_STREAM_PAUSE_RESUME(Data 1) kein andere Sensor Befehl wie (ROOMBA_SENSORS oder ROOMBA_QUERY_LIST gesetzt worden sein!
Parameter
cKommando das gesendet werden soll
Rückgabe
Mögliche Rückgabe Werte sind:
  • 0 kein Fehler
  • -ROOMBA_ERROR_TO_BIG Angebe Gruppen überschreiten das Maximum von ROOMBA_MAX_DATA Sensor Daten
  • -ROOMBA_ERROR_AGAIN Keine SensorDaten vorhanden! Wird nur zurück geben wenn ROOMBA_NONBLOCKING gesetzt ist!
